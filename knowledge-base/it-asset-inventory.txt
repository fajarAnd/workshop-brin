2. ASSET CATEGORIZATION & TAGGING
=============================================================================

Asset Tag Format: BRIN-[CATEGORY]-[YEAR]-[SEQUENCE]

Category Codes:
- DT: Desktop Computer
- LT: Laptop
- SV: Server (Physical)
- NT: Network Equipment
- ST: Storage Equipment
- PR: Printer/Scanner
- MB: Mobile Device
- AC: Accessories (Monitor, Keyboard, Mouse)

Contoh Asset Tag:
- BRIN-DT-2024-0234: Desktop tahun 2024, sequence 234
- BRIN-LT-2023-0567: Laptop tahun 2023, sequence 567
- BRIN-SV-2022-0012: Server tahun 2022, sequence 12

Asset Tag Location:
- Laptop: Sticker di bagian belakang layar (dekat BRIN logo)
- Desktop: Sticker di side panel casing (visible saat di meja)
- Server: Sticker di front panel (visible di rack)
- Network Equipment: Sticker di top panel device

4. ASSET REQUEST & PROCUREMENT PROCESS
=============================================================================

A. NEW ASSET REQUEST

Step 1: Justification & Approval
- User mengisi form request: https://assetrequest.brin.go.id
- Sertakan business justification:
  * Kebutuhan bisnis/research
  * Alasan tidak bisa menggunakan aset existing
  * Urgency level
  * Budget code/cost center
- Approval workflow:
  * Department Head → Approval dalam 3 hari kerja
  * IT Manager → Review spesifikasi, approval dalam 2 hari
  * Finance → Budget verification, approval dalam 2 hari
  * Procurement → Process PO dalam 5 hari

Step 2: Procurement
- IT Procurement team proses pembelian
- Vendor selection (berdasarkan e-catalog/tender)
- Purchase Order (PO) issued
- Estimasi delivery: 2-4 minggu untuk hardware standar
- Estimasi delivery: 6-12 minggu untuk server/specialized equipment

Step 3: Receiving & Asset Tagging
- Asset diterima di IT warehouse
- Quality inspection & testing
- Asset tagging (barcode & RFID)
- Registration di Asset Management System (ServiceNow)
- Photos dokumentasi

Step 4: Deployment & Handover
- IT team install OS & mandatory software
- Security baseline configuration
- User training (jika diperlukan)
- User sign asset acknowledgment form
- Asset assigned di system

Total Process Time: 4-8 minggu (dari request sampai deployment)

---

B. ASSET REPLACEMENT (Refresh/Upgrade)

Kriteria Asset Eligible untuk Replacement:
- Umur ≥ 4 tahun (laptop) atau ≥ 5 tahun (desktop)
- Performance degradation signifikan (tidak support aplikasi terkini)
- Biaya maintenance > 40% harga baru
- End of vendor support/warranty
- Hardware failure berulang (>3x dalam 6 bulan)

Replacement Cycle Priority:
1. Laptops: 4 tahun
2. Desktops: 5 tahun
3. Servers: 7 tahun
4. Network equipment: 7 tahun
5. Printers: 5 tahun

Process:
- IT Asset team monitor aging assets via dashboard
- Generate replacement plan annually (setiap Oktober untuk tahun berikutnya)
- Priority diberikan berdasarkan:
  * Business criticality
  * Performance issues
  * User tier (researchers > admin staff)
- Approval di annual IT budget planning
- Deployment dilakukan bertahap (quarterly batches)

---

C. ASSET RETURN & DISPOSAL

Asset Return Scenarios:
- Employee resignation/termination
- Employee transfer department
- Asset malfunction (RMA/replacement)
- Project completion (temporary assignment)

Return Process:
1. User submit return request via https://assetreturn.brin.go.id
2. IT Admin verifies asset condition
3. Data wipe (secure erase) - mandatory untuk semua storage devices
4. Asset status updated: RETURNED
5. Asset options:
   - Reassign to new user (jika kondisi baik)
   - Send untuk repair/refurbish
   - Mark untuk disposal (jika end-of-life)

Asset Disposal (End-of-Life):
- Kriteria: Umur >7 tahun, tidak ekonomis untuk repair, obsolete technology
- Process:
  1. IT Asset Manager approval untuk disposal
  2. Finance team verify depreciation complete
  3. Data destruction (secure wipe + physical destruction untuk sensitif data)
  4. Certificate of Destruction issued
  5. Asset removed dari inventory system
  6. Disposal via authorized vendor (e-waste recycling) atau trade-in program

Data Security saat Disposal:
- Mandatory secure erase dengan DoD 5220.22-M standard (minimum 3-pass)
- Hard drives/SSD untuk high-security data: Physical destruction (shredding/degaussing)
- Certificate of Data Destruction disimpan 5 tahun untuk audit

5. ASSET TRACKING & AUDIT
=============================================================================

Asset Management System:
- Platform: ServiceNow IT Asset Management (ITAM)
- Access: https://assetmanagement.brin.go.id
- Features:
  * Real-time inventory tracking
  * Asset lifecycle management
  * Contract & warranty management
  * Cost tracking & depreciation
  * Compliance reporting

Asset Tracking Methods:
- Barcode scanning (semua assets)
- RFID tags (high-value assets >Rp 10 juta)
- Software agent (untuk laptops/desktops - auto-inventory)
- Network discovery (untuk network equipment & servers)

Physical Audit Schedule:
- High-value assets (>Rp 50 juta): Quarterly
- Standard assets: Semi-annually
- Complete inventory audit: Annually (setiap Desember)
- Spot checks: Random monthly (10% sample)

Audit Process:
1. Generate audit list dari Asset Management System
2. Physical verification team scan barcode/RFID
3. Verify asset location, condition, user assignment
4. Identify discrepancies:
   - Missing assets → Investigation & report
   - Unauthorized assets → Registration atau removal
   - Mismatched location → Update system
5. Audit report to IT Manager & Finance
6. Discrepancy resolution plan (30 hari)

Missing Asset Protocol:
- User wajib report missing asset dalam 24 jam
- IT Security investigation (CCTV review, access logs)
- Police report (untuk theft >Rp 10 juta atau sensitif data)
- Insurance claim (jika applicable)
- User liability assessment (negligence vs theft)

6. ASSET MAINTENANCE & SUPPORT
=============================================================================

Preventive Maintenance Schedule:

Desktops/Laptops:
- Every 6 months: Clean dust, check thermal paste, update firmware
- Every 12 months: Full hardware diagnostics, replace thermal paste (laptop)

Servers:
- Every 3 months: Firmware update, clean dust filters, check fan operation
- Every 6 months: Full hardware diagnostics, battery backup test (RAID controllers)
- Every 12 months: Replace batteries (RAID, BIOS), deep cleaning

Network Equipment:
- Every 6 months: Firmware update, clean dust, check port status
- Every 12 months: Cable management review, port utilization analysis

UPS:
- Every 3 months: Battery test
- Every 12 months: Battery replacement (untuk critical systems)
- Every 24 months: Capacitor inspection

Warranty & Support Contracts:

Standard Warranty Coverage:
- Desktops/Laptops: 3 years on-site NBD (next business day)
- Servers: 5 years 24/7 with 4-hour response SLA
- Network Equipment: Cisco SmartNet 8x5xNBD
- Enterprise Servers: 24/7 with 2-hour response SLA

Extended Warranty Options:
- Accidental Damage Protection (ADP): For laptops (recommended untuk field researchers)
- Battery replacement service: For laptops >2 years old
- Extended warranty: Additional 2 years beyond standard (untuk critical systems)

Support Escalation:
- Level 1: IT Support helpdesk (internal)
- Level 2: IT Engineering team (internal)
- Level 3: Vendor technical support
- Level 4: Vendor engineering escalation (critical issues)
